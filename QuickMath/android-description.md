# QuickMath — Android Platform Description

## Overview

The Android target is one of four platforms supported by the MAUI Blazor project. It runs the shared Blazor UI inside a native `BlazorWebView` hosted in a `MauiAppCompatActivity`.

---

## Configuration

### AndroidManifest.xml
**Location:** `Platforms/Android/AndroidManifest.xml`

```xml
package="com.pikmintea.quickmath"
label="Quickmath"
```

| Property | Value |
|----------|-------|
| Package name | `com.pikmintea.quickmath` |
| App label | `Quickmath` |
| `allowBackup` | `true` |
| `supportsRtl` | `true` |
| Icon | `@mipmap/appicon` |

**Permissions declared:**
- `android.permission.ACCESS_NETWORK_STATE` — check connectivity
- `android.permission.INTERNET` — network access (required for Blazor WebView)

**Notable missing permissions (for future features):**
- No camera, microphone, location, storage, or notification permissions declared

### MainActivity.cs
**Location:** `Platforms/Android/MainActivity.cs`

```csharp
[Activity(
    Theme = "@style/Maui.SplashTheme",
    MainLauncher = true,
    ConfigurationChanges = ConfigChanges.ScreenSize
        | ConfigChanges.Orientation
        | ConfigChanges.UiMode
        | ConfigChanges.ScreenLayout
        | ConfigChanges.SmallestScreenSize
        | ConfigChanges.Density
)]
```

- Inherits `MauiAppCompatActivity`
- `MainLauncher = true` — this is the entry point activity
- `ConfigurationChanges` handles screen rotation, resizing, dark/light mode switches, and density changes without restarting the activity
- No overrides — entirely default behavior

### MainApplication.cs
**Location:** `Platforms/Android/MainApplication.cs`

- `[Application]` attribute marks it as the Android `Application` class
- Delegates app creation to `MauiProgram.CreateMauiApp()`
- Standard boilerplate, no custom initialization

## Potential Issues & Risks

### 1. Package Name Mismatch
**Severity: Low**

The `.csproj` declares:
```xml
<ApplicationId>com.companyname.quickmath</ApplicationId>
```

But `AndroidManifest.xml` declares:
```xml
package="com.pikmintea.quickmath"
```

The manifest `package` attribute takes precedence. This inconsistency can cause confusion during publishing, Firebase project linking, or Play Store registration. **Recommendation:** Align both to the same value.

---

### 2. Google Auth on Android (Not Configured)
**Severity: High (if auth is needed)**

`Google.Apis.Auth` is installed but:
- No SHA-1 fingerprint has been configured in any Firebase console
- No `google-services.json` file exists in the Android platform folder
- No auth service registration in `MauiProgram.cs`
- Google Sign-In on Android requires a properly configured OAuth client linked to the app's signing certificate

**To fix:** Add `google-services.json` to `Platforms/Android/`, configure Firebase project, and register SHA-1 fingerprints.

---

### 3. Firestore on Android (Not Configured)
**Severity: High (if backend is needed)**

`Google.Cloud.Firestore` is installed but:
- No Firestore initialization code exists
- No service classes for data access
- All Home page data is hardcoded
- Firestore on MAUI Android requires proper Firebase setup (same as auth)

---

### 4. WebView Performance & Compatibility
**Severity: Medium**

- `BlazorWebView` on Android uses the system WebView (Chromium-based). Android 7.0 (API 24) ships with an older WebView that may lack support for modern CSS/JS features
- The min SDK of API 24 means the app could run on devices with WebView versions that don't fully support .NET 10 Blazor's JavaScript interop requirements
- **Recommendation:** Consider raising min SDK to API 26+ for better WebView support, or test thoroughly on API 24-25

---

### 5. Color Theme Inconsistency
**Severity: Low-Medium**

- `colors.xml` uses `#512BD4` (default .NET purple)
- App CSS theme uses `#8e24ff` (lighter purple)
- The splash screen and status bar during launch will show a different shade than the running app
- **Recommendation:** Update `colors.xml` to match the app's accent color (`#8e24ff` / `#6a1bcc` for dark variant)

---

### 6. No ProGuard/R8 Configuration
**Severity: Medium (for release builds)**

- No `proguard.cfg` or R8 rules file exists
- BlazorWebView and AOT-compiled .NET code may get incorrectly stripped in Release builds
- When building for release, add appropriate keep rules for MAUI and Blazor types

---

### 7. No Network Security Config
**Severity: Medium (if using local dev server)**

- No `network_security_config.xml` defined
- If the app needs to communicate with a local development server (HTTP), Android 9+ (API 28+) blocks cleartext traffic by default
- The `INTERNET` permission is declared, but cleartext is not explicitly allowed
- **To fix:** Add a `network_security_config.xml` to allow cleartext for development domains, or ensure all traffic uses HTTPS

---

### 8. No Hardware Acceleration Explicitly Set
**Severity: Low**

- `BlazorWebView` benefits from hardware acceleration for smooth rendering
- The manifest doesn't explicitly set `android:hardwareAccelerated="true"` on the `<application>` tag
- It's enabled by default on API 14+, but being explicit avoids edge cases

---

### 9. ConfigurationChanges May Mask Issues
**Severity: Low**

The `ConfigurationChanges` flags prevent the activity from being recreated on rotation, but:
- The BlazorWebView may not properly handle all configuration changes without a refresh
- Some resources (like CSS media queries for orientation) won't re-evaluate until the WebView is notified
- **Recommendation:** Test rotation thoroughly; consider removing `ConfigChanges.UiMode` if dark/light mode switching needs to trigger a proper theme reload

---

### 10. No Android-Specific Permissions for Future Features
**Severity: Informational**

The following permissions are NOT declared but may be needed later:
- `CAMERA` — for profile photos or AR features
- `READ_EXTERNAL_STORAGE` / `READ_MEDIA_IMAGES` — for image picking
- `POST_NOTIFICATIONS` (API 33+) — for streak reminders
- `VIBRATE` — for haptic feedback on correct/wrong answers
- `WAKE_LOCK` — if preventing screen sleep during practice sessions

---

### 11. Debug Profile is Device-Specific
**Severity: Low**

The `.csproj.user` references:
- `Fairphone FP4 (Android 15.0 - API 35)` as active debug profile
- `pixel_7_-_api_36_0` as default device

These are machine-specific entries (likely from the developer's workstation). They won't affect other developers but will cause the IDE to show warnings if those devices aren't available.

---

### 12. No Multi-ABI Configuration
**Severity: Medium (for distribution)**

The `.csproj` doesn't specify `<RuntimeIdentifiers>` for Android:
- Default will build for the host architecture only during debug
- For release/APK distribution, you'll need to specify `android-arm;android-arm64` at minimum
- Without this, the published APK may not work on all devices

---

### 13. Dotnet Bot Image Still Present
**Severity: Informational**

`dotnet_bot.svg` is included in `Resources/Images/` and referenced in the `.csproj` with a `BaseSize` of 168x208. This is a default template asset that should be removed or replaced.

---

### 14. No AndroidX or Play Services Dependencies Explicitly Declared
**Severity: Low**

MAUI 10 bundles most AndroidX libraries, but if Google Auth or Firestore are integrated later, explicit AndroidX and Play Services package references may be needed to avoid version conflicts.

## Build & Publish Commands

### Debug on Android (emulator)
```bash
dotnet build -t:Run -f net10.0-android
```

### Debug on connected device
```bash
dotnet build -t:Run -f net10.0-android --device
```

### Publish APK (debug)
```bash
dotnet publish -f net10.0-android -c Debug
```

### Publish AAB (release, for Play Store)
```bash
dotnet publish -f net10.0-android -c Release
```

Output location: `bin/Release/net10.0-android/publish/`
